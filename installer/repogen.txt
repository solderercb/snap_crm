Usage: repogen.exe [options] repository-dir

Options:
  -p|--packages dir         The directory containing the available packages.
                            This entry can be given multiple times.
  --repository dir          The directory containing the available repository.
                            This entry can be given multiple times.
  -e|--exclude p1,...,pn    Exclude the given packages.
  -i|--include p1,...,pn    Include the given packages and their dependencies
                            from the repository.
  --ignore-translations     Do not use any translation
  --ignore-invalid-packages Ignore all invalid packages instead of aborting.
  --ignore-invalid-repositories Ignore all invalid repositories instead of aborting.
  -s|--sha-update p1,...,pn List of packages which are updated using
                            content sha1 instead of version number.
  -r|--remove               Force removing target directory if existent.
  --update                  Update a set of existing components (defined by
                            --include or --exclude) in the repository
  --update-new-components   Update a set of existing components (defined by
                            --include or --exclude) in the repository with all new components
  -v|--verbose              Verbose output
  --unite-metadata          Combine all metadata into one 7z. This speeds up metadata
                            download phase.
  --component-metadata      Creates one metadata 7z per component.
  --af|--archive-format 7z|qbsp
                            Set the format used when packaging new component data archives. If
                            you omit this option the 7z format will be used as a default.
  --ac|--compression 0,1,3,5,7,9
                            Sets the compression level used when packaging new data archives.

Example:
  repogen.exe -p ../examples/packages repository/

Примечания
  Опция --update, согласно документации https://doc.qt.io/qtinstallerframework/ifw-updates.html
    нужна для того, чтобы утилита при создании репозитория не пересоздавала архивы с неизменёнными файлами.
    В примечании сказано:
      repogen recreates the 7zip archives each time it is being called. As 7zip stores the timestamps 
      of the included files (which are moved or copied during this process), the SHA sum of each 
      archive changes. SHA sums are used to verify the download of the archive and hence the SHA needs 
      to match the 7zip. As the SHAs are stored in the Updates.xml file you will be forced to upload 
      the full repository. This can be circumvented by using the --update option of repogen.
    Если я всё правильно понял, то при каждом запуске утилиты исходные файлы копируются во временное 
    хранилище и их метки времени изменяются. 7zip сохраняет метки времени файлов, следовательно SHA 
    контрольные суммы архивов будут изменяться, а это приведёт к тому, что архивы нужно будет каждый 
    раз загружать на сервер целиком (а они могут быть большими). И чтобы этого избежать, используется 
    опция --update.
    На практике, даже без этой опции пересозданные архивы с компонентами имеют идентичные контрольные 
    суммы, изменяется только файл с мета-информацией и Updates.xml (касается IFW версии 4.6.0 с моими 
    модификациями; поведение оригинальной версии не исследовано).
  Назначение опции --update-new-components не ясно, разницы в работе с опцией --update за время 
    экспериметов не заметил.
  При необходимости частичного обновления компонентов программы, следует добавить пакет с новым 
    названием и по сути выполнить процедуру добавления или изменения компонентов в maintenancetool.
    Если создать архив для обновления, в котором будут только новые файлы (для экономии траффика), то 
    maintenancetool при обновлении удалит все старые файлы. Т. е. механизм работает по принципу 
    удаление-установка.
  Чтобы новый пакет компонентов автоматически устанавливался при выполнении обновления, неоходимо в
    файле package.xml добавить <Essential>true</Essential>.
  опциями -e и -i указываются названия пакетов (т. е. папок); не архивов и не отдельных файлов.
